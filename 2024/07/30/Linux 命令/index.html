tail

用于在命令行中查看文件的末尾若干行，通常用于查看文件的最后几行内容，特别是日志文件或者其他追加内容。

  • 选项：指定输出的格式

常用选项

选项 作用
-n [K] 显示文件的前K行，默认前10行

-F 类似于-f，但在文件旋转时尝试跟踪新文件

-q 多个文件时不输出文件名

举例

[!注]

  1. 文件旋转指的是什么？
    指的是在持续写入日志或其他数据的文件时，定期或根据特定条件对文件进行重命名、压缩或清理的过程。这个过程的目的是管理文件大小，避免文件变得过大而影响系统性能或占用过多存储空间。

  2. --follow=name\|descriptor的使用：

    • name：跟踪文件名，原文件重命名或移动之后，不再跟踪原文件，跟踪新的同名文件。
    • descriptor：跟踪文件描述符，即使原文件重命名或者移动，仍然跟踪原文件。

关于Linux 的文件权限

在Linux 中，将文件操作对象分为 3 类，即文件所有者、群组和其他对象，对于某一文件，这 3 类对象分别有不同的操作权限。

其中权限又被分为读(r)、写(w)和执行(x)权限，因此对于一个文件，每类用户权限都有 8 种状态，可以使用 3 个八进制数来代表 3 类用户的权限。例如 777 表示 rwxrwxrwx，因此 chmod 777 [文件名] 表示设置某文件对所有用户都具有读写和执行权限。

grep

用于文件中的搜索匹配的文本，支持正则表达式并打印匹配的行。

  • 选项：匹配的规则（可缺省）
  • 模式：匹配的文本、正则表达式等
  • 文件名：支持多个文件，在展示匹配结果的时候，会显示文件名

常用选项

选项 作用
-i 忽略大小写
-v 反转匹配，显示不包含模式的行

-W 匹配整个单词
-x 匹配整行
–color 输出中高亮显示匹配的文本

[!注意]

  1. 正则表达式：指的是一种用来描述字符串模式的表达式，常用于文本处理中。正则表达式语法
    • ^pattern：用于匹配 pattern 为首的行。
    • pattern$：用于匹配 pattern 结尾的行。
    • p.ttern：.表示任意字符。
  2. 管道符：grep 通常用于结合管道符来使用，将一个命令的输出作为 grep 的匹配对象。
    • ps -aux | grep "nginx"：系统进程，筛选出包含 “nginx” 的行。
    • tail file.log | grep "error" | sort：日志文件，筛选出 “error” 的行，并排序。
    • grep "error" /var/log/syslog | grep "$(date +"%b %e %H" -d "1 hour ago")"：过滤出系统日志中的错误消息，然后使用当前时间减去1小时的格式来进一步过滤1小时内的错误消息。

awk

用于文本处理和数据分析，具有较强的灵活性和文本处理能力，适合处理结构化数据，例如表格、日志文件等。

  • 模式：用于匹配行，支持文本、正则表达式、关系运算符和BEGIN/END等（可缺省，默认匹配所有行）
  • 动作：用于指定匹配成功后的操作，可以是多个命令组成的语句块
  • 文件名：支持多个文件，在展示匹配结果的时候，不会显示文件名

常用选项

选项 作用

模式 举例 作用

动作 举例 作用

字段和记录

awk将每一行称为一条记录，每一列成为一个字段，默认情况下，字段使用空格或者制表符分隔，可以使用 -F 选项自己指定分隔符。

  • $0，$1，$2，$3...：0表示整行，其他数字分别表示具体字段。
  • NF：当前行的字段数。
  • NR：当前记录的行数。

内置变量和函数

  1. 变量
    • FS：输入字段分隔符（默认空格/制表符）。
    • OFS：输出字段分隔符（默认空格）。
    • RS：输入记录分隔符（默认换行符）。
    • ORS：输出记录分隔符（默认换行符）。
  2. 函数
    • length(s)：返回字符串长度。
    • substr(s, start, length)：返回字符串从 start 位置开始的子串，长度为 length。
    • toupper(s)：将字符串转换为大写。
    • tolower(s)：将字符串转换为小写。
    • sin(x)、cos(x)、sqrt(x)、log(x)：数学相关函数。
    • systime()：返回当前时间戳。

控制结构

  1. awk支持常见的控制结构，条件控制和循环控制：
    • 条件：if (condition) {action} else {action}
    • 循环：while (condition) {action} for (i = 0; i < n; ++i) {action}

  1. BEGIN 和 END
    • 在 awk 中，BEGIN 块在处理任何输入行之前执行，而 END 块在处理完所有输入行之后执行。由于在 BEGIN 块执行时，还没有读取任何输入数据，因此在BEGIN中使用 $1 $2 是未定义行为，打印时将输出空值。同样，在 END 块之后不能再通过 {} 打印其他输出。

    • END 块中的代码在所有输入文本行处理之后执行。通常用于打印总结信息或清理工作。

sort

用于文件内容排序的命令行工具，对文本中的行进行排序，提供多种选项控制排序行为。

  • 选项：控制排序行为的选项（可缺省）

常用选项

选项 作用
-b 忽略每行前面的空白字符
-d 使用字典序

-r 逆序排序

[!注]

  1. 关于 -k 的用途
    • 首先如果没有指定分隔符，默认使用 " " 作为分隔符。
    • -k，--key=POS1[,POS]：表示排序的字段从 POS1 到 POS2 ，如果只指定了 POS1 ，表示从该字段开始，到行结束。
    • sort -k 1,2 text.txt 表示按照第1字段到第2字段（包含第1和第2字段）排序。首先根据第1字段排序，如果第1字段相同，则根据第2字段排序。
    • 其中 POS 后还可以添加一些选项来对某些特定字段进行特殊化排序。
    • sort -k 1,1 -k 2nr,2nr text.txt 表示第1字段按照字典序排序，第2字段按照数值逆序排序。
  2. 关于 -t 和 -k 的结合使用
    • -t 用来设置分隔符号，-k用来指定具体的排序区域。

权限描述

权限描述分为数字形式和符号形式，数字形式使用三个8进制数字来表示三种不同文件访问对象的权限，在此不过多赘述，主要谈谈符号形式。

符号形式使用四种符号来描述访问对象：

  • u：表示文件所有者。
  • g：表示组成员对象。
  • o：表示其他成员对象。
  • a：涵盖了u、g、o。

使用三种权限来描述不同的文件访问权限：

  • r：表示读权限。
  • w：表示写权限。
  • x：表示执行权限。

使用三种符号来描述权限操作：

  • +：为某文件==追加==某权限。
  • -：为某文件==减少==某权限。
  • =：将某文件==设置==为某权限。

-L

-m

-p

-s

-u

-U

userdel

  • 选项：（可缺省）如果缺省，则会保留用户目录和用户数据
  • 用户名：需要删除的用户

常用选项

选项 作用
-r
–remove		 删除用户主目录和内容
-f
–force		 强制删除用户，即使用户已经登录或运行进程
-Z
–selinux-user		 指定删除用户时的SELinux用户标签

[!注]
如果系统使用SELinux，可以使用-Z指定在删除用户时应用的SELinux用户标签，这是为了确保在删除用户时也正确更新SELinux上下文。

chage

用于修改和查看用户密码的有效期信息。可以帮助管理员控制密码过期时间和用户需要修改密码的频率。

  • 选项：
  • 用户名：需要修改密码配置的用户

常用选项

选项 作用
-l 显示用户密码的有效期信息
-d 设置用户上次更改密码的日期
-E 设置用户账户过期日期
-I 设置密码过期后多少天账户被禁用
-m 设置密码最短使用期限(多少天内不能更改)
-M 设置密码最长使用期限(多少天后必须更改)
-W 设置密码过期前多少天警告用户
-h 显示帮助信息

id

id命令常用于显示用户和组的信息。可以显示当前用户或者指定用户的UID，GID和用户所属的附加组。

  • 选项：如果缺省，则显示全部信息（可缺省）
  • 用户名：如果缺省，则显示当前用户UID和GID（可缺省）

常用选项

选项 作用
-u 显示用户UID
-g 显示用户基本组GID
-G 显示用户所属附加组GID
-n 显示用户和组的名字，而不是ID
-r 显示真实用户ID，而不是有效用户ID
-Z 显示SELinux上下文

groupadd

  • 选项：（可缺省）
  • 组名：新创建的组名

常用选项

选项 作用

-r 创建一个系统组，GID小于1000
-o 允许使用重复的GID

gpasswd

用于管理组密码文件，并进行组管理员相关操作。

  • 选项：对组的一些操作（可缺省）
  • 用户名：只针对组操作时可缺省
  • 组名：需要操作的组名，当前两项都缺省时，可以修改组密码

常用选项

选项 作用
-a
–add		 添加用户到组
-d
–delete		 从组中删除用户
-r
–remove-password		 删除组密码
-R
–restrict		 禁止组成员自行切换到组
-A
–administrators		 设置组管理员
-M
–members		 定义组成员

[!注]

  1. 什么是禁止组成员自行切换到组？
    一般来说用户可以通过 newgrp [组名] 来切换到另一个组，从而影响用户对文件和目录的访问权限，当通过 gpasswd -R [组名] 命令之后，非组成员用户无法通过 newgrp [组名] 来切换到该组。

groupmod

  • 组名：需要操作的组名

常用选项

选项 作用

-n 修改组名
-o 允许使用重复的GID
