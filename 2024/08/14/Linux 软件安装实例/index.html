问题

systemd不能启动Redis

如果遇到这样的错误，可以在命令行中测试启动Redis。

如果遇到这样的错误，表示Redis 服务器尝试使用 systemd 进行监督，但编译时未启用 libsystemd 支持。Redis 在编译时缺少 systemd-devel 组件。

解决方法：安装 systemd-devel 组件并重新编译安装

编译安装

cmake

如果需要使用 systemd 来管理 MySQL 那就需要在编译时设置相应参数。

  • DBUILD_CONFIG：使用官方推荐的配置
  • DCMAKE_INSTALL_PREFIX：设置安装路径
  • DCPACK_MONOLITHIC_INSTALL：让 make package 生成单个安装文件
  • DWITH_DEBUG：禁用 DEBUG
  • DWITH_SSL：支持 SSL
  • DWITH_BOOST：boost 安装目录
  • DWITH_SYSTEMD：表示使用 systemd 来管理 MySQL 服务
  • DSYSTEMD_PID_DIR：指定 PID 文件存储路径

boost

运行 cmake3 命令时可能会提示找不到 boost ，我们可以通过命令获取 boost 相应版本，这里我们需要选择 boost_1_77_0.tar.bz2。
