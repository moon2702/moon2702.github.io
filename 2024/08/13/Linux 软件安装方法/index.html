Linux 软件安装方法

发表于2024-08-13|学习经历

编译安装

编译安装分为 5 部分，配置编译安装依赖，下载源码并解压，生成配置，编译源码，复制可执行文件到对应目录可执行文件

最重要的就是三个步骤：

  • ./configure：通过选项传递安装参数构建Makefile 文件，会参考用户输入和Makefile.in 文件，例如安装路径、启用特性等；检查依赖的外部环境，例如依赖的软件包。
  • make：根据Makefile 文件，检测依赖环境，构建应用程序。
  • make install：复制可执行文件到指定目录。

安装前准备

在安装前可以通过查看README，INSTALL来获取帮助。

在安装前需要配置一些开发工具和环境来进行辅助编译：

  • 开发工具：make，gcc（c/c++编译器）
  • 开发环境：开发库（glibc：标准库），头文件，开发组包Development Tools
  • 软件相关依赖包

编译安装

configure

运行configure 脚本，生成Makefile 文件，主要功能是：

  • 指定安装位置
  • 指定启用的特性

选项介绍：

选项 作用
–prefix= 指定默认安装位置，默认为/usr/local/
–sysconfidr= 指定配置文件的位置
system types 支持交叉编译
–bindir= 指定可执行文件的位置
–sbindir= 指定系统管理命令的位置
–libdir= 指定库文件的位置
–includedir= 指定头文件的位置
–datadir= 指定数据文件的位置
–localstatedir= 指定可变数据文件（日志等）的位置
–enable- 启用某些功能或选项 --enable-shared 启用共享库
–disable- 禁用某些功能或选项 --disable-static 禁用静态库
–with-<PACKAGE[=ARG]> 启用特定的包或者库 --with-ssl 启用SSL支持
–without- 禁用特定的包或者库 --without-ssl 禁用SSL支持
–help 查看帮助

注意：常用被编译操作依赖的开发组件程序，通常安装组件程序本身 name ，还需要安装 name-devel。
