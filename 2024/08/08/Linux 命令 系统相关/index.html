文件系统

文件系统指的是操作系统用来管理和组织存储设备上数据的方法和结构，不同文件系统适用于不同的场景。

  1. EXT4：主要用于Linux 系统，支持大容量和高性能
  2. NTFS：Windows 系统主要的文件系统，支持大文件和高安全性
  3. FAT32：通用的文件系统，兼容性好，适合移动存储设备
  4. APFS：苹果系统的文件系统，支持快照、加密和高性能
  5. tmpfs：临时内存文件系统，用于临时数据文件，速度快、对系统影响小

du

–time 显示每个文件或者目录的修改时间

文件大小

在文件系统中，文件的大小分为实际文件大小和占用空间大小，在磁盘中文件系统以固定的大小分配一块存储空间（例如4KB为一个块），如果一个文件小于以管块的大小，则会占用整个块的大小，即使这个文件只占用10个字节数，我们可以称这10个字节为该文件的实际文件大小，4KB为该文件的占用空间大小。

  • 实际文件大小：文件的实际大小，即文件中的字节数，用来反映文件中的数据总量。
  • 占用空间大小：文件占用磁盘空间的大小，通常以磁盘块的大小作为单位，考虑到了内存碎片等因素，用来反映磁盘空间的占用。

top

用于动态显示系统中运行的进程信息和系统资源的使用情况，提供一个交互式、实时刷新的界面，监控系统的运行状态。还有一个功能相似的软件htop ，需要自行安装，功能相较top 更加丰富。

功能

  1. 显示系统总体信息：例如运行时间、用户数量、平均负载、物理内存和交换空间等使用情况。
  2. 提供进程列表：显示当前运行的所有进程。包括进程ID、用户名、状态(运行或者睡眠)、CPU使用率、内存使用率等信息。
  3. 交互式操作：通过键盘进行交互式操作。例如 h 显示帮助信息、k 结束进程、esc 退出当前交互界面、q 退出top 界面等。
  4. 排序和过滤：通过不同按键修改进程排序方式。例如 P 按照CPU 使用情况排序、M 按照内存使用情况排序、u 只显示特定用户的进程。
  5. 设置显示模式：例如自定义显示列。可以通过命令行参数或者top 界面按下 O 来设置。

vmstat

用于报告系统虚拟内存、进程、CPU活动、I\O操作等方面的统计信息。帮助用户了解系统的性能状况，例如诊断内存、CPU或者I\O相关问题时很有用。

常用选项

选项 作用
-a 显示活跃和非活跃内存的信息
-s 显示系统自启动以来的累积信息统计
-d 显示磁盘相关统计信息
-p 显示指定分区的统计信息
-f 显示fork的次数
-m 显示内存页面的大小
-t 在报告中显示时间戳
-w 在报告中显示交换活动
-S 设置内存显示单位 (K、M 表示1024) (k、m表示1000)

使用 vmstat 命令之后，显示下列数据：

  • system：系统信息

ping

用于测试与目标主机之间网络连通性的工具，通过ICMP协议来实现。

常用选项

选项 作用
-c 指定发送ICMP的次数
-i 设置ICMP包的时间间隔
-s 设置ICMP包的数量
-t 设置ICMP包的生存时间(TTL)
-q 以静默方式启动，仅显示结果
-w 设置超时时间，最长等待时间

traceroute

用于追踪网络连接路径，显示从源主机到目标主机之间经过的每一跳路由器的信息，通过发送TTL值逐渐增加的ICMP或者UDP数据包来确定每一跳路由器。

常用选项

选项 作用
-n 不解析主机名，直接显示IP
-m 设置最大跳数，默认30
-q 每个节点发送的探测数据包数量，默认3
-w 每个回复等待时间，默认5s
-I 使用ICMP作为回显请求，默认UDP
-T 使用TCP的SYN数据包
-p 设置起始端口号，默认33434
-f 设置第一个数据包的TTL值，默认为1

锁定内存页

在计算机内存管理中，“页被锁在内存中”是指一个内存页被标记为不能被操作系统的虚拟内存管理机制交换（或“换出”）到硬盘上的交换空间中。这意味着该页将始终驻留在物理内存中，不会被换出。

锁定内存页通常是为了确保关键数据始终在物理内存中可用，而不会因为内存压力或其他原因被换出。主要用于以下几种场景：

  • 实时的应用程序：需要保证低延迟和高响应的应用程序。
  • 内核模块和驱动程序：需要直接访问物理内存的模块。
  • 安全性：防止敏感数据被写入到交换空间中，降低数据泄漏的风险。
  • DMA(直接内存访问)：使用DMA的设备需要锁定内存，这些设备直接与物理内存交互，不经过CPU。

如何锁定内存页面？

通过 mlock 和 mlockall 系统调用来锁定内存页面。

kill

通过发送信号来控制进程行为，默认情况下，发送 SIGTERM 信号，请求进程正常终止。

常用信号

信号 字段 作用
-1 SIGHUP 挂起进程，通常用于重启进程
-9 SIGKILL 强制终止进程，无法被进程捕获或忽略，立即停止进程
-15 SIGTERM 请求进程终止，运行进程进行清理操作
-18 SIGCONT 恢复已经暂停的进程
-19 SIGSTOP 暂停指定进程
